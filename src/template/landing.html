Rapid IoC PHP Development Framework

If you're seeing this page, it means you've correctly setup this framework.

Annotation powered framework?

I love how Spring MVC used annotations in Java, and wondered why something similar hasn't been built in PHP, with the world moving to a minimal code approach (and with good reason) I've decided to build this framework. Power your application with very little and self explanatory code.

I've also ensured that this framework has little to zero overhead on your application, as frameworks are usually the likely culprits for sluggish performance.

Getting started with CryoCore is so easy! If you're seeing this page it means that the controller LandingController.cryo.php exists, so you can check the source code there. But just for the sake of showing how simple it is to setup a simple webpage with CryoCore, I'm going to post below!

src/MyController.cryo.php - Notice the .cryo.php? this is essential! 

<?php
    namespace App; // important, anything outside this namespace will not be recognised

    @Controller 
    class MyController {

        @Get( path="/home" , produces="application/json" )
        public function showHomepage(){
            return "I Work";
        }
    }
?>

Now heading to http://localhost/home , you should see a text document saying "I work", this is all that's needed to create a basic controller. Each controller can have multiple routes



<?php
    namespace App; // important, anything outside this namespace will not be recognised

    @Controller 
    class MyController {

        @Get( path="/home" , produces="text/text" )
        public function showHomepage(){
            return "I Work";
        }
        @Get( path="/home/page2" , produces="text/html" )
        public function showHomepage(){
            return "<p>I'm a p tag</p>";
        }
    }
?>

All controllers are autoloaded, you don't need to include them or require them. Just add a @Controller annotation and let Cryo do the work!

Why use CryoCore?

I believe that setting up an API or Application should be much more simple then it is. With other frameworks you have to make sure you have composer installed any use bulky libraries for primitive functionality, this is slow and adds a lot of technical debt to your application.

I designed this library with minimalism in mind while making sure you've still got tonnes of functionality. I take advantage of PHP's core mechanisms and caching to ensure lightning fast performance.
